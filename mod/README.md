# WtgArchipelago — MelonLoader mod

The in-game client that connects **WHAT THE GOLF?** to an Archipelago multiworld:
detects level clears (sends checks) and applies received items (unlocks chambers,
grants Flags). Pairs with the `what_the_golf` apworld in the parent repo.

> **Why MelonLoader, not BepInEx?** BepInEx's Dobby runtime-invoke detour
> **hard-crashes this game** at Unity engine init (reproducible with zero
> plugins). MelonLoader uses a different bootstrap and runs the game fine.

## Confirmed working

- Engine: **Unity 2020.3.48f1, IL2CPP**. Loader: **MelonLoader v0.7.3**.
- The mod loads and all three Harmony hooks bind at runtime (verified in-game):
  `Il2CppCore.Level:Complete`, `Il2CppCore.Level:Fail`,
  `GameAnalytics:OnFinalBossCompleted`.
- AP client: **Archipelago.MultiClient.Net** 6.6.0.

## Il2CppInterop naming (important)

MelonLoader's interop **prefixes namespaced game types with `Il2Cpp`**:
`Core.Level` → **`Il2CppCore.Level`**, `Core.LevelData` → `Il2CppCore.LevelData`.
Global-namespace types keep their names: `GameAnalytics`, `LevelManager`.

## Layout

```
WtgArchipelago.csproj   net6.0 x64; references MelonLoader\net6 + Il2CppAssemblies
src/
  Mod.cs                MelonMod entry (OnInitializeMelon applies patches; OnUpdate = pump)
  Plugin.cs             loader-agnostic holder + LogAdapter (wraps MelonLogger)
  Archipelago/          ArchipelagoClient, ArchipelagoData, DeathLinkHandler
  Mapping/              LocationMap, ItemApplier, GameState
  Patches/GamePatches.cs Harmony hooks (real Il2Cpp names)
ids.json                generated by ../tools/export_ids.py (AP name -> id)
```

## Build & deploy

```
python ../tools/export_ids.py   # regenerate ids.json from the apworld
dotnet build -c Debug           # builds + auto-deploys (see below)
```

The build **auto-deploys**:
- the mod → `<game>\Mods\WtgArchipelago.dll` (MelonLoader loads mods from the
  **game-root** `Mods\`, not `MelonLoader\Mods\`).
- Its dependency `Archipelago.MultiClient.Net.dll` is staged in `<game>\UserLibs\`.

> **Kill the game before rebuilding** — a running instance locks the deployed DLL
> and the post-build copy will fail.

Launch via Steam; logs at `<game>\MelonLoader\Latest.log`.
